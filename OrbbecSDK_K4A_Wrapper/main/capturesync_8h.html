Capturesync module asynchronously accepts new captures from color and depth modules through this API.

Parameters
capturesync_handleThe capturesync handle from capturesync_create()
resultThe result of the USB opperation providing the sample.
capture_rawA capture of a single color image, or a capture with upto 2 images; depth and IR.
color_captureTrue if the capture contains a single color capture. False is the capture contains upto 2 images; depth and IR.
Remarks
If K4A_SUCCEEDED(result) is true then capture_raw must be valid. If K4A_SUCCEEDED(result) is false then capture_raw is optional and may be NULL.

◆ capturesync_destroy()

void capturesync_destroy ( capturesync_t  capturesync_handle)

Destroys an capturesync instance

Parameters
capturesync_handleThe capturesync handle to destroy

This call only cleans up the capturesync handle. This function should not be called until all outstanding k4a_capture_t objects are freed.

This function should return 0 to indicate the number of outstanding allocations. Consider this the number of leaked allocations.
