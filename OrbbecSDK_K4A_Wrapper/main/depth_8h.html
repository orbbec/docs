◆ depth_cb_streaming_capture_t

typedef void() depth_cb_streaming_capture_t(k4a_result_t result, k4a_capture_t capture_handle, void *callback_context)

Delivers a sample to the registered callback function when a capture is ready for processing.

Parameters
resultindicates if the capture being passed in is complete
capture_handleCapture being read by hardware.
callback_contextContext for the callback function
Remarks
Capture is only of one type. At this point it is not linked to other captures. Capture is safe to use during this callback as the caller ensures a ref is held. If the callback function wants the capture to exist beyond this callback, a ref must be taken with capture_inc_ref().

Open a handle to the depth device.

Parameters
depthmcu_handle[IN] Handle to the MCU communication module
calibration_handle[IN] Handle to the calibration module.
capture_ready_cb[IN] A function pointer to call when new depth captures are ready
capture_ready_cb_context[IN] A context to go with the depth_capture_ready callback function
depth_handle[OUT] A pointer to write the opened depth device handle to
Returns
K4A_RESULT_SUCCEEDED if the device was opened, otherwise an error code

If successful, depth_create will return a depth device handle in the depth parameter. This handle grants exclusive access to the device and may be used in the other k4a API calls.

When done with the device, close the handle with depth_destroy
