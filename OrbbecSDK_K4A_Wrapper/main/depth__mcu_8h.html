◆ depthmcu_stream_cb_t

typedef void() depthmcu_stream_cb_t(k4a_result_t result, k4a_image_t image_handle, void *callback_context)

Delivers a sample to the registered callback function when a capture is ready for processing.

Parameters
resultindicates if the capture being passed in is complete
image_handleImage being read by hardware.
callback_contextContext for the callback function
Remarks
Capture is only of one type. At this point it is not linked to other captures. Capture is safe to use durring this callback as the caller ensures a ref is held. If the callback function wants the capture to exist beyond this callback, a ref must be taken with capture_inc_ref().

◆ depthmcu_create()

k4a_result_t depthmcu_create ( uint32_t  device_index,
depthmcu_t *  depthmcu_handle 
)

Open a handle to the depth mcu device.

Parameters
depthmcu_handle[OUT] A pointer to write the opened depthmcu device handle to
Returns
K4A_RESULT_SUCCEEDED if the device was opened, otherwise an error code

If successful, depthmcu_create will return a depthmcu device handle in the depth mcu parameter. This handle grants exclusive access to the device and may be used in the other k4a API calls.

When done with the device, close the handle with depthmcu_destroy
